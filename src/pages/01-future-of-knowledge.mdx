## 引言：当笔记不再只是文字

互联网的伟大发明之一是**超链接**——它让知识从孤岛变成了网络。但六十多年过去了，大部分"知识库"本质上依然只是带有超链接的静态文本。

想象一下：如果一篇笔记里不仅能写文字，还能嵌入**实时数据图表**、**可交互代码沙盒**、**智能卡片**和**沉浸式词条解释**——知识会怎样呈现？

这就是 **Digital Garden** 正在探索的未来。

## 组件三：沉浸式词条悬浮

在传统文章中，术语解释要么放在文末、要么需要读者自行搜索。现在，<HoverDetail term="数字花园" href="https://en.wikipedia.org/wiki/Digital_garden">
  数字花园（Digital Garden）是一种非线性、持续生长的个人知识管理范式。与传统的"博客"不同，数字花园中的内容不按时间排序，而是像植物一样自由生长、交叉关联、不断修剪。
</HoverDetail> 这个概念可以通过悬浮组件即时解释。

再比如，当你在笔记中提到 <HoverDetail term="组件化思维" href="https://react.dev/learn/thinking-in-react">
  组件化思维（Component Thinking）源自软件工程中的组件设计模式。它将复杂系统拆解为独立、可复用的单元，每个单元封装自己的状态、样式和行为——这与知识管理中将概念模块化、可组合的理念不谋而合。
</HoverDetail>，读者不需要翻到附录——信息就在指尖。

> **设计哲学**：好的知识库应该减少「认知摩擦」，让读者始终保持流动的阅读状态，而不是在页面之间反复跳转。

## 组件四：可编辑代码沙盒

理论讲再多，不如动手试一试。下面的 `LiveCodeBlock` 组件允许你在笔记内部直接编写和运行 HTML/JS 代码：

<LiveCodeBlock
  title="试一试：一个极简计数器"
  code={`<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body {
      display: flex;
      flex-direction: column;
      align-items: center;
      justify-content: center;
      min-height: 100vh;
      font-family: system-ui, sans-serif;
      background: #fafafa;
    }
    .counter {
      font-size: 64px;
      font-weight: 600;
      color: #18181b;
      margin-bottom: 24px;
      font-variant-numeric: tabular-nums;
    }
    button {
      padding: 10px 28px;
      font-size: 16px;
      border: none;
      border-radius: 12px;
      background: #3b82f6;
      color: white;
      cursor: pointer;
      transition: all 0.2s ease;
    }
    button:hover {
      background: #2563eb;
      transform: scale(1.04);
    }
    button:active {
      transform: scale(0.96);
    }
  </style>
</head>
<body>
  <div class="counter" id="counter">0</div>
  <button onclick="increment()">+1</button>
  <script>
    let count = 0;
    function increment() {
      count++;
      document.getElementById('counter').textContent = count;
    }
  </script>
</body>
</html>`}
  template="static"
  title="交互演示：点击按钮试试 👆"
  previewHeight={320}
/>

你可以修改上面的代码，比如把 `+1` 改成 `+5`，把颜色改成你喜欢的样子——修改生效实时可见。

## 结语：知识库的第四范式

回顾知识管理的演化：

1. **纸笔时代**：线性记录，难以检索
2. **超文本时代**：Wiki 和博客，静态链接
3. **富媒体时代**：Notion/Obsidian，嵌入多媒体但缺乏交互
4. **超级组件时代**：**数据、代码、可视化与交互深度融合**——笔记本身成为应用

Digital Garden 的目标不是再做一个博客框架。它想回答的问题是：

**当知识的载体不再是静态文档，而是一个图灵完备的交互环境时——** 知识的表达力将获得怎样的解放？
